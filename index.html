Multi-Agent RL for Schiphol Airport Cargo

Can autonomous agents learn to coordinate truck slot allocation at one of Europe's busiest cargo hubs — without a central controller?

Amsterdam Airport Schiphol's cargo hub operates without a unified slot booking system, leading to dock congestion during peak periods. This project builds a full discrete-event simulation of the landside cargo ecosystem using SimPy, wraps it as a PettingZoo multi-agent environment, and trains MAPPO agents via BenchMARL and TorchRL. Three agent types interact — a Transporter (51 actions), four GHAs with shared policy (193 actions each), and an Orchestrator acting last (251 actions) — under a custom α-blended reward structure that balances private efficiency with global hub throughput. The environment enforces real DTP slot booking rules including frozen windows, priority windows, ANPR gate checks, and race condition guards across simultaneous agent actions.

Key engineering decision

Bridging SimPy's continuous event loop with PettingZoo's discrete step interface required a custom "Time-Warping Window" macro-step pattern. At each RL step, SimPy advances by a fixed Δt, flushes the sensor log, and only then emits rewards and observations — keeping the MDP well-defined without halting the simulation clock.

02

Ishihara CNN — Color Blindness Recognition

Ishihara charts encode digits through color cluster distributions, not strokes. Can a compact CNN learn color relationships well enough to classify them?

A PyTorch CNN trained on 1,400 auto-generated Ishihara chart images to classify digits 0–9. The task is fundamentally different from MNIST — the model must learn spatial color distributions rather than stroke geometry. The project runs a structured experiment log across 13 runs, testing scheduler strategies (ReduceLROnPlateau vs CosineAnnealingLR vs none), resolution variants (24×24 to 256×256), and augmentation techniques. Final architecture uses three Conv blocks with BatchNorm, AdaptiveAvgPool2d for resolution independence, and a two-layer FC classifier — 129,290 trainable parameters total.

Counterintuitive finding

Lower resolution (24×24) outperformed 128×128 and 256×256 inputs, reaching 99.6% test accuracy on 280 samples. Because AdaptiveAvgPool2d collapses all spatial information before classification, higher resolution added compute cost without adding signal to the classification head — and smaller spatial maps forced the filters to learn color aggregation more directly.

03

Ames House Price Prediction — ML from Scratch

What happens when you build the entire ML pipeline — from missing data imputation to cross-validated regression — without touching scikit-learn?

A full data science pipeline on the Ames Housing dataset implemented in raw NumPy and Pandas. Every component is written from scratch: chi-squared MCAR tests, one-hot and ordinal encoding, min-max normalization, stratified K-Fold CV with quantile binning, OLS and Ridge regression via normal equations, gradient descent linear regression with early stopping, and Lasso via coordinate descent. Four dataframe variants (original, log, winsorized, IQR-capped) are evaluated across all models to measure the effect of outlier handling strategy on performance.

Why build from scratch

The goal was not to maximize RMSE but to build a precise, first-principles understanding of the pipeline. Knowing exactly what stratification does during CV, why Ridge shrinks differently from Lasso, and where preprocessing leakage enters the pipeline matters more for long-term depth than calling sklearn.fit().
